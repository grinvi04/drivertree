## 1. 프로젝트 개요

### 1.1 한 줄 정체성
**"면허 학원도, 유튜브도, 공식 사이트도 알려주지 않는, 초보운전자가 진짜 막막한 순간들을 정리한 가이드"**

### 1.2 만들게 된 배경
운전면허 취득과 실주행 초기 과정에서 정보의 빈 공간이 명확히 존재한다.

- **공식 채널 (도로교통공단 등)**: 정보는 정확하지만 어렵고 공식적이라 초보 입장에서 이해가 힘들다.
- **비공식 채널 (커뮤니티, 블로그)**: 접근성은 좋지만 정보가 부정확하거나 의견이 엇갈린다.
- **유튜브**: 조회수 위주의 신차 리뷰·브이로그가 대부분이고, 진짜 필요한 실용 정보가 적다.
- **면허 학원**: 시간 압박으로 인해 페달 밟기·미러 보기 같은 기초조차 제대로 안 가르치는 경우가 많다.

이 빈 공간을 메우는 것이 본 프로젝트의 목적이다.

### 1.3 미션
초보운전자가 처한 **단계와 상황에 맞춰**, **공식 정보의 정확성**과 **실생활 적용의 현실성**을 함께 제공하여, 운전이라는 새로운 영역에 진입하는 사람들의 막막함을 줄인다.

### 1.4 프로젝트 목적
- **실사용자 서비스** — 실제 초보운전자가 검색하고 도움받는 서비스
- **풀스택 아키텍처** — NestJS + Next.js + PostgreSQL 통합 운영
- **데이터 기반 개선** — Vercel Analytics + Sentry 기반 사용자 행동 분석 및 서비스 개선

## 2. 페르소나

### 2.1 P1 — 예비 운전자 (면허 취득 고민·준비 중)
- **상황**: 면허 학원 등록 여부, 비용, 기간을 비교하며 시작점을 찾고 있다.
- **고민**: "학원 vs 독학 어떤 게 나아?", "총 비용이 얼마나 들지?", "독학은 정말 가능한가?"
- **검색 키워드**: "운전면허 학원 비용", "독학으로 면허", "1종 보통 vs 2종"

### 2.2 P2 — 실주행 1년 미만 초보 운전자
- **상황**: 면허는 땄지만 실제 도로에서 마주치는 상황마다 헷갈리거나 자신 없다.
- **고민**: "이 표지판 진짜 의미가 뭐지?", "이 상황에서 누가 양보해야 해?", "지금 내가 한 게 위반인가?"
- **검색 키워드**: "비보호 좌회전", "골목길 마주침", "어린이보호구역 단속"

### 2.3 P3 — 차를 처음 다루는 사람
- **상황**: 운전은 어느 정도 익숙해졌지만 차 자체에 대한 지식이 부족하다.
- **고민**: "주유소에서 뭘 어떻게 해야 하지?", "세차장 종류와 차이?", "차 고장 났을 때 누구 부르지?", "사고 났을 때 절차?"
- **검색 키워드**: "셀프 주유 방법", "자동세차 vs 손세차", "교통사고 처리 절차"

## 3. 핵심 차별점 (Why Us)

### 3.1 단계별 흐름 기반 콘텐츠
- 시중 사이트는 **주제별**로만 정리되어 있다.
- 본 사이트는 **취득 전 → 취득 → 실주행 → 차량 관리**라는 **사용자 단계별**로 구조화한다.

### 3.2 공식 정보와 실생활 매너의 통합
- 법규는 **공식 데이터 기반의 정확성**을 보장한다.
- 동시에 **실제 도로의 운전 매너**를 솔직하게 함께 다룬다.

### 3.3 실용·생활밀착형 콘텐츠
- 차 외적인 영역(주유, 세차, 고장, 사고)을 함께 다룬다.

### 3.4 AI 챗봇 기반 상황별 답변
- 사용자가 **상황을 자연어로 설명**하면, AI 챗봇이 관련 콘텐츠를 검색해 답변한다.
- 기존 키워드 검색의 한계를 극복한다.

## 4. 사용자 시나리오 (User Story)

### S1. 면허 학원 등록을 앞둔 사람
> 학원 등록을 앞두고 비용과 절차를 알아보기 위해 사이트를 방문한다. 메인 페이지의 "면허 취득 가이드" 카테고리에서 **학원 vs 독학 비교표**를 보고, 자신의 상황에 맞는 선택을 한다. 학원 등록 시 챙겨야 할 준비물 체크리스트를 북마크한다.

### S2. 주행 중 모르는 상황을 만난 운전자
> 좁은 골목에서 차와 마주쳤다. 한 대씩 비켜야 하는데 누가 먼저 양보해야 하는지 모르겠다. 사이트의 AI 챗봇에 "골목길에서 차랑 마주쳤어요"라고 입력하면, 관련 법규 + 매너 + 사례를 정리한 답변이 나온다. 본문 콘텐츠로 이동해 더 자세한 내용을 확인한다.

### S3. 첫 사고를 겪은 운전자
> 가벼운 접촉 사고가 발생했다. 보험사를 부르기 전에 무엇을 해야 하는지 몰라 사이트에 접속한다. "사고 대처" 카테고리에서 단계별 절차를 확인하고, **범칙금 계산기**로 본인 케이스의 예상 처분을 미리 가늠한다.

### S4. 첫 차 구매를 앞둔 사람
> 차를 사면 매달 얼마가 드는지 가늠이 안 된다. **차량 유지비 계산기**에 차종·연식·예상 주행거리를 입력하면, 보험료·세금·유류비·정비비를 합한 월 평균 유지비가 계산되어 나온다.

## 5. MVP 기능 명세

### F1. 단계별 콘텐츠 카탈로그
**설명**: 사이트의 본질. 사용자가 단계·카테고리별로 콘텐츠를 탐색할 수 있다.

**카테고리 구조**:
1. **면허 취득 가이드** — 학원 vs 독학, 비용, 절차, 시험 준비
2. **운전 기본기** — 페달, 미러, 핸들링, 주차 기초
3. **도로 법규·신호** — 표지판, 신호체계, 실제 도로의 매너
4. **차량 관리·생활** — 주유, 세차, 고장 대처, 정기 점검
5. **사고·이슈 대처** — 사고 절차, 보험, 범칙금/과태료

**기능 요구사항**:
- 카테고리·태그·키워드 필터링
- 콘텐츠 상세 페이지 (이미지·동영상 임베드 가능)
- 관련 콘텐츠 자동 추천 (같은 태그 기반)
- SEO 최적화 (URL slug, OG 태그, JSON-LD)

### F2. AI 챗봇 ★ 킬러 기능
**설명**: AI 챗봇. 사용자가 자연어로 상황을 묘사하면 관련 콘텐츠를 검색해 답변한다.

**기능 요구사항**:
- 사이트 내 콘텐츠를 벡터 DB(pgvector)에 인덱싱
- 사용자 질문 → 유사 콘텐츠 검색 → TF-IDF + 코사인 유사도로 답변 생성
- 답변에 출처 콘텐츠 링크 함께 표시 (신뢰도 확보)
- 대화 히스토리 유지 (세션 단위)
- 부적절한 질문 필터링 (프롬프트 인젝션 방어)

**비기능 요구사항**:
- 답변 평균 응답 시간 3초 이내
- Rate Limit: 분당 10회 / 시간당 60회

### F3. 상황·키워드 통합 검색
**설명**: 키워드 + 자연어 혼합 검색.

**기능 요구사항**:
- 키워드 검색 (PostgreSQL Full-Text Search)
- 자연어 검색 (벡터 유사도 검색)
- 검색 결과 카테고리·태그별 필터
- 검색어 자동완성 (인기 검색어 기반)

### F4. 관리자 백오피스
**설명**: 운영자가 콘텐츠를 작성·수정·삭제할 수 있는 관리자 페이지.

**기능 요구사항**:
- 관리자 로그인 (단일 관리자 계정, JWT 인증)
- 콘텐츠 CRUD (마크다운 에디터 + 이미지 업로드)
- 카테고리·태그 관리
- 챗봇 사용 로그 모니터링 (질문·답변·평가)
- 콘텐츠 발행 상태 (초안 / 발행 / 보관)

### F5. 범칙금·과태료 계산기
**설명**: 위반 항목을 선택하면 예상 범칙금/과태료/벌점을 계산해주는 도구.

**기능 요구사항**:
- 위반 항목 선택 (속도 위반, 신호 위반, 주차 위반 등 주요 30~50개)
- 차종(승용차/승합차/이륜차) 및 단속 환경(어린이 보호구역 등) 입력
- 결과: 범칙금 / 과태료 / 벌점 + 관련 법규 콘텐츠 링크
- **데이터 출처 명시 필수** (도로교통공단 공식 자료 기반)
- 면책 조항 표시 ("실제 처분과 다를 수 있음")

### F6. 차량 유지비 계산기
**설명**: 차종·연식·주행거리·운전 패턴을 입력하면 월 평균 유지비를 추정해주는 도구.

**기능 요구사항**:
- 입력 항목: 차종, 연식, 연료 종류, 월 예상 주행거리, 보험 등급 등
- 계산 항목: 유류비 / 자동차세 / 보험료(추정) / 정기 점검비 / 소모품 교체비
- 결과: 월 평균 유지비 + 항목별 비중 차트
- **추정값임을 명시** (실제 비용은 사용자 상황에 따라 다름)

## 6. 비기능 요구사항

### 6.1 성능
- 첫 페이지 로드 (LCP) 2.5초 이내
- API 응답 시간 평균 200ms 이내 (챗봇 제외)
- 검색 응답 시간 1초 이내

### 6.2 SEO
- 모든 콘텐츠 페이지 SSR 또는 SSG
- 메타 태그, OG 태그, JSON-LD 구조화 데이터
- sitemap.xml 자동 생성
- robots.txt 최적화

### 6.3 접근성
- 모바일 우선 반응형 (운전자가 모바일로 자주 접근)
- WCAG AA 수준의 기본 접근성 (alt 텍스트, 키보드 네비게이션)

### 6.4 보안
- 관리자 페이지 인증 보호
- API Rate Limiting
- 콘텐츠 입력 XSS 방어
- LLM 프롬프트 인젝션 방어

### 6.5 운영
- 모든 환경변수 `.env` 분리
- 에러 로깅 (Sentry 또는 자체 로깅)
- 챗봇 LLM 호출 비용 모니터링

## 7. 콘텐츠 전략

### 7.1 초기 시드 콘텐츠 분량
- 5개 카테고리 × 평균 20~30개 = **총 100~150개**
- 우선순위: 면허 취득 가이드 > 도로 법규 > 차량 관리 > 사고 대처 > 운전 기본기

### 7.2 콘텐츠 작성 방식
- AI로 초안 생성 (도로교통공단 공식 자료를 컨텍스트로 제공)
- **본인이 반드시 검수** (법규 정확성은 100% 사람 검증 필요)
- 톤: 초보운전자가 이해할 수 있는 친근한 문체

### 7.3 데이터 소스 및 저작권
- **도로교통공단·경찰청·국토교통부** 공식 자료를 참고하되, **직접 인용은 명시**하고 사용자 친화 톤으로 재작성
- 공식 자료의 PDF·이미지를 그대로 가져오지 않음 (저작권 이슈)
- 본인 경험·매너 콘텐츠는 자체 작성

- 운전자 커뮤니티 / Q&A 게시판

## 9. 성공 지표

### 9.1 정량 지표 (1차 MVP 출시 후 3개월 목표)
- 월 방문자(MAU) **1,000명** 이상
- 평균 페이지 체류 시간 **2분** 이상
- AI 챗봇 일일 평균 질의 수 **20건** 이상
- Google PageSpeed 점수 (모바일) **80점** 이상

### 9.2 정성 지표
- 사용자 피드백을 통해 콘텐츠의 정확성·실용성 검증
- AI 챗봇 답변의 만족도 (사용자 평가 기능 추가 후 측정)

### 9.3 포트폴리오 지표 (취업 활용 관점)
- 면접에서 본 프로젝트에 대해 **자신 있게 답변할 수 있는 디테일** 확보
- `docs/AI_LOG.md`를 통한 AI 활용 사례의 구체적 시연
